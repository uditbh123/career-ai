# ADR-001: Select FastAPI as the Backend Framework

**Status:** Accepted

**Date:** 2026-07-10

**Decision Makers:** Career AI Core Team

## Context

Career AI requires a backend capable of exposing REST APIs, handling authentication, integrating with AI providers, communicating with external job portals, and managing persistent user data.

The backend must support modern Python development practices, asynchronous programming, automatic API documentation, and long-term maintainability.

This framework is expected to remain the foundation of the application for several years.

---

# Problem Statement

Select a backend framework that provides:

* High performance
* Strong developer experience
* Excellent type safety
* Automatic API documentation
* Async support
* Easy testing
* Large community support
* Long-term maintainability

---

# Architectural Drivers

The following priorities influenced this decision:

1. Maintainability
2. Scalability
3. Developer productivity
4. Strong typing
5. Performance
6. Modern Python ecosystem
7. Easy onboarding for contributors

---

# Alternatives Considered

## Option 1 — FastAPI

Modern Python framework built on Starlette and Pydantic.

### Advantages

* Excellent performance (ASGI)
* Native async support
* Automatic OpenAPI documentation
* Strong type checking
* Built-in request validation
* Excellent IDE support
* Easy dependency injection
* Growing ecosystem
* Well suited for AI applications

### Disadvantages

* Smaller ecosystem than Django
* Fewer batteries-included features
* Async programming introduces additional complexity

---

## Option 2 — Flask

Lightweight WSGI framework.

### Advantages

* Very mature
* Huge ecosystem
* Easy learning curve
* Large community

### Disadvantages

* No built-in validation
* No automatic API documentation
* Limited async capabilities
* Requires many additional extensions

---

## Option 3 — Django

Full-stack web framework.

### Advantages

* Extremely mature
* Excellent admin interface
* Built-in authentication
* Large ecosystem
* Production proven

### Disadvantages

* Heavy framework for an API-first architecture
* Includes many features unnecessary for this project
* More opinionated
* Higher complexity for our use case

---

# Decision

Career AI will use **FastAPI** as its backend framework.

---

# Rationale

Career AI is designed as an API-first platform rather than a traditional server-rendered web application.

FastAPI aligns with this architecture by providing:

* excellent async performance
* automatic OpenAPI documentation
* modern type-safe development
* seamless integration with AI services
* simple request validation
* excellent developer experience

Because the frontend is completely separated from the backend, we do not require Django's templating system or administrative interface.

FastAPI provides the best balance between simplicity, scalability, and modern Python practices.

---

# Consequences

Positive:

* Faster development
* Cleaner API architecture
* Automatic Swagger documentation
* Better developer experience
* Strong typing throughout the codebase
* Easier testing

Negative:

* Smaller ecosystem than Django
* Some developers may be unfamiliar with async programming
* Certain third-party packages are designed primarily for Flask or Django

---

# Risks

* Incorrect async implementation could reduce performance.
* Heavy business logic inside API endpoints would reduce maintainability.
* Overusing dependency injection could make the code harder to understand.

---

# Risk Mitigation

* Keep API endpoints thin.
* Place business logic inside services.
* Use async only where it provides measurable benefit.
* Follow FastAPI best practices and official documentation.

---

# Migration Strategy

If FastAPI no longer satisfies project requirements:

1. Preserve business logic within the Service Layer.
2. Preserve database access inside the Repository Layer.
3. Replace only the API layer.
4. Maintain API contracts where possible.
5. Minimize disruption to the rest of the system.

---

# Review Criteria

This decision should be revisited if:

* FastAPI is no longer actively maintained.
* Performance requirements change significantly.
* A different architectural style is adopted.

---

# Decision Outcome

Accepted.

FastAPI best satisfies the current and anticipated requirements of Career AI while maintaining flexibility for future growth.
