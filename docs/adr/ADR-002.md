# ADR-002: Select PostgreSQL as the Primary Database

**Status:** Accepted

**Date:** 2026-07-10

**Decision Makers:** Career AI Core Team

## Context

Career AI manages structured information including users, profiles, education, work experience, resumes, applications, companies, interviews, and AI-generated content.

The system requires strong data consistency, transactional integrity, complex relationships, and reliable querying.

---

# Problem Statement

Choose a database capable of supporting long-term application growth while maintaining reliability and performance.

---

# Architectural Drivers

* ACID compliance
* Relational data
* Scalability
* Strong SQL support
* Mature ecosystem
* Docker compatibility
* ORM compatibility

---

# Alternatives Considered

## PostgreSQL

### Advantages

* Excellent relational database
* ACID compliant
* Advanced indexing
* JSON support
* Full-text search
* Mature ecosystem
* Excellent SQLAlchemy support
* Widely used in production

### Disadvantages

* Slightly steeper learning curve
* More operational complexity than SQLite

---

## MySQL

### Advantages

* Mature
* Large community
* High performance
* Excellent tooling

### Disadvantages

* Less advanced SQL features
* Historically weaker JSON functionality
* Some ORM features are PostgreSQL-first

---

## SQLite

### Advantages

* Extremely simple
* Zero configuration
* Great for prototypes

### Disadvantages

* Not suitable for concurrent production workloads
* Limited scalability
* Not appropriate for multi-user applications

---

## MongoDB

### Advantages

* Flexible schema
* Fast document storage
* Easy horizontal scaling

### Disadvantages

* Weak fit for highly relational data
* Complex joins
* Increased application complexity
* Potential data duplication

---

# Decision

Career AI will use PostgreSQL as its primary database.

---

# Rationale

Most data within Career AI is relational.

Examples include:

* User → Profile
* User → Experience
* User → Applications
* Job → Company
* Resume → Skills
* Interview → Application

These relationships naturally fit a relational database model.

PostgreSQL additionally provides JSON support, allowing future AI metadata and embeddings to be stored without abandoning relational integrity.

---

# Consequences

Positive:

* Excellent scalability
* Reliable transactions
* Strong relational modeling
* Excellent ORM integration

Negative:

* More setup than SQLite
* Requires database administration knowledge

---

# Risks

* Poor schema design could impact performance.
* Missing indexes may slow queries.
* Incorrect migrations may corrupt production data.

---

# Risk Mitigation

* Use Alembic migrations.
* Review schema changes.
* Add indexes only after measurement.
* Back up production data before migrations.

---

# Migration Strategy

Business logic must never depend directly on PostgreSQL-specific features unless absolutely necessary.

Repositories isolate database operations, allowing migration to another relational database if required.

---

# Review Criteria

Revisit if:

* Project requirements become predominantly document-oriented.
* PostgreSQL no longer meets scaling requirements.
* Operational costs outweigh benefits.

---

# Decision Outcome

Accepted.

PostgreSQL provides the best combination of reliability, scalability, relational modeling, and ecosystem support for Career AI.
