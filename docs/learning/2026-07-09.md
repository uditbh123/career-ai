# Learning Log – Day 1

## What I built

Today I started my main portfolio project, Career AI. Instead of jumping straight into coding features, I focused on creating a professional project foundation. I created the GitHub repository, initialized the project structure, organized the backend and frontend directories, added documentation folders, configured the `.gitignore`, created a virtual environment, and wrote the initial README.

## What I learned

One of the biggest lessons today was that software development starts long before writing business logic. A well-organized project structure makes the codebase easier to understand, maintain, and scale.

I also learned why Python virtual environments are important. They isolate project dependencies so different projects on the same computer don't interfere with each other.

Another important lesson was understanding why the backend and frontend are separated. Although they are part of the same application, they have different responsibilities, technologies, and deployment processes. Keeping them separate allows each part to evolve independently, makes collaboration easier, and creates a cleaner architecture.

I also learned that good Git commit messages and documentation are just as important as writing code because they communicate the project's history and design decisions to other developers.

## Challenges

At first, creating folders without writing any actual application code felt unnecessary. However, I realized that experienced software engineers invest time in planning because it reduces problems later in development.

Another challenge was resisting the urge to start coding immediately. Learning to build software methodically instead of rushing into implementation is a new mindset for me.
